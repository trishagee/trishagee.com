I'm a Java Developer. &nbsp;But I'm also a Web Developer. &nbsp;Web Developers have been so badly maligned over the last decade or so that I always feel wary (and sometimes slightly ashamed) admitting this. &nbsp;There's some sort of assumption that Web Developers (and Front End Developers) aren't real programmers. Similarly, "real" developers don't like to be tainted by coming into contact with that nasty "front end stuff" in case someone mistakes them for a designer.<br /><div><br /></div><div>Trust me, no-one is going to mistake a Java Developer for a designer. &nbsp;For a start, when designers wear geeky glasses it's ironic. &nbsp;Or chic. &nbsp;Or something.</div><div><br /></div><div>But developers will be forced to do something around the front end at some point in their lives. &nbsp;Even if it's because they're sick of manually kicking off some process and want to give the users a big red button to press instead.</div><div><br /></div><div>So, as much to compensate for my own goldfish-like brain as anything else, I'm going to make a note of some of the CSS stuff I've used or found helpful during my mad scramblings to get the <a href="http://www.lmaxtrader.co.uk/">LMAX Trader</a> user interface to a) look the way we wanted b) perform fast enough so our <a href="http://www.infoq.com/presentations/LMAX">awesome back end performance</a> wasn't totally invisible to the retail user and c) Not Suck Too Badly in Internet Explorer.</div><div><br /></div><div><h3>Part One: Horizontal Layout (or: There's No Excuse For Tables Any More)</h3></div><div>You may (or may not) have heard that using tables for layout is a Bad Thing. &nbsp;But to be fair, most of us don't care. &nbsp;I've done it myself, it's usually the quickest way to lay stuff out on the page, especially if you're new to HTML / CSS and/or short on time. &nbsp;There are loads of arguments all over the web as to why this is a bad thing, but the reasons I didn't want tables on the <a href="http://www.lmaxtrader.co.uk/">LMAX</a> UI was a) we saw the performance improve by a really simple replacement of tables with divs and b) given the amount the UI was mutating, divs positioned with CSS was going to allow us a much quicker turnaround for the umpteenth re-design of the UI (ideally the upshot of this would be letting the designers mess with the CSS when they changed their minds and leave us developers out of it).</div><div><br /></div><div>Anyway let's assume Tables Are Bad. &nbsp;Tables are only for tabular data, not for layout. &nbsp;In my world.<br /><br /></div><div><h4>1.1 Simple Horizontal Layout Using Inline</h4></div><div>The most common use of tables for layout is where you put elements side by side - the default behaviour of divs is that they lay out underneath each other.</div><div><br /></div><div>However it's pretty easy to get divs to behave this way too, and using divs has some advantages over using tables.</div><div><div class="separator" style="clear: both; text-align: center;"><a href="http://2.bp.blogspot.com/_7adQd_YVgmQ/TTQhm7XQHTI/AAAAAAAAH7c/3JE1vQBCbgw/s1600/Screen+shot+2011-01-14+at+17.14.37.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img /></a></div></div><blockquote><pre><code>&lt;html&gt;<br />&lt;head&gt;<br />    &lt;title&gt;Horizontal flow&lt;/title&gt;<br />    &lt;style type="text/css"&gt;<br />        #left {<br />            background-color: cyan;<br />            display: inline;<br />        }<br />        #center {<br />            background-color: yellow;<br />            display: inline;<br />        }<br />        #right {<br />            background-color: red;<br />            display: inline;<br />        }<br />    &lt;/style&gt;<br />&lt;/head&gt;<br /><br />&lt;body&gt;<br />&lt;div id="container"&gt;<br />    &lt;div id="left"&gt;Left&lt;/div&gt;<br />    &lt;div id="center"&gt;Center&lt;/div&gt;<br />    &lt;div id="right"&gt;Right&lt;/div&gt;<br />&lt;/div&gt;<br />&lt;/body&gt;<br />&lt;/html&gt;</code></pre></blockquote>The first option is to use <code style="font-size: 12px;">display: inline</code>. &nbsp;This will lay div elements next to each other, similar to a <code style="font-size: 12px;">span</code>. &nbsp;This is the simplest way to get divs to appear side-by-side.<br /><div class="separator" style="clear: both; text-align: left;"><br /></div><div class="separator" style="clear: both; text-align: left;">The disadvantage of this technique, however, is that inline elements can't be sized the same way as standard divs. &nbsp;By default they contract to fit the content.</div><div class="separator" style="clear: both; text-align: left;"><br /></div><div class="separator" style="clear: both; text-align: left;">The screenshot above is from Chrome on the mac - you'll notice by default the elements have some spacing between them. &nbsp;Reducing the margin/borders on the elements doesn't seem to eliminate this, so that's something else you might need to consider if you use this mechanism to display elements side-by-side.</div><div class="separator" style="clear: both; text-align: left;"><br /></div><div class="separator" style="clear: both; text-align: left;"></div><div><h4>1.2 Simple Horizontal Layout Using Floats</h4></div><div class="separator" style="clear: both; text-align: center;"><a /></a></div><div><blockquote><pre><code><br />&lt;html&gt;<br />&lt;head&gt;<br />    &lt;title&gt;Horizontal flow&lt;/title&gt;<br />    &lt;style type="text/css"&gt;<br />        #left {<br />            background-color: cyan;<br />            float: left;<br />        }<br />        #center {<br />            background-color: yellow;<br />            float: left;<br />        }<br />        #right {<br />            background-color: red;<br />            float: left;<br />        }<br />    &lt;/style&gt;<br />&lt;/head&gt;<br /><br />&lt;body&gt;<br />&lt;div id="container"&gt;<br />    &lt;div id="left"&gt;Left&lt;/div&gt;<br />    &lt;div id="center"&gt;Center&lt;/div&gt;<br />    &lt;div id="right"&gt;Right&lt;/div&gt;<br />&lt;/div&gt;<br />&lt;/body&gt;<br />&lt;/html&gt;<br /></code></pre></blockquote></div><div>Similar to using inline, floating a div will do the following:<br /><ol><li>Make it shrink to fit the content</li><li>Lay it out side-by-side with any sibling floating divs.</li></ol>You'll want to apply padding, margins, height, width and whatever else to make it appear less rubbish.<br /><div><h4><br class="Apple-interchange-newline" />1.3 Wrapping Horizontal Layout</h4></div><div class="separator" style="clear: both; text-align: center;"><a /></a></div><div><blockquote><pre><code><br />&lt;html&gt;<br />&lt;head&gt;<br />    &lt;title&gt;Horizontal flow&lt;/title&gt;<br />    &lt;style type="text/css"&gt;<br />        #left {<br />            background-color: cyan;<br />            width: 50%;<br />            float: left;<br />        }<br />        #center {<br />            background-color: yellow;<br />            width: 50%;<br />            float: left;<br />        }<br />        #right {<br />            background-color: red;<br />            width: 50%;<br />            float: left;<br />        }<br />    &lt;/style&gt;<br />&lt;/head&gt;<br /><br />&lt;body&gt;<br />&lt;div id="container"&gt;<br />    &lt;div id="left"&gt;Left&lt;/div&gt;<br />    &lt;div id="center"&gt;Center&lt;/div&gt;<br />    &lt;div id="right"&gt;Right&lt;/div&gt;<br />&lt;/div&gt;<br />&lt;/body&gt;<br />&lt;/html&gt;<br /></code></pre></blockquote></div>The advantage you have using divs over tables is that you can get the browser to work out how the elements should flow. &nbsp;Tables require you to determine exactly how many cells appear on each row, but with divs you can determine a width for each element and have the browser work out whether to show it on a new line or not. &nbsp;This will work with both percentage and pixel widths. <br /><ul><li>If you want a more table-like structure where you know exactly how many elements should appear on each line, use a percentage of the screen with. &nbsp;</li><li>If you know the width of each element, you should set a pixel width on the elements and have the browser work out how many to show per line.</li></ul><br />The next CSS post will go over floating behaviour in a little more detail.</div>
